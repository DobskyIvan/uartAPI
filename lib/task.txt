 

На выполнение тестового задания 5 дней, 2 дня на ревью и предоставление обратной связи.

 

Напишите, пожалуйста, если у Вас есть вопросы по заданию.

 

Тестовое задание

 

Шаг 1

Написать публичный API для взаимодествия с блоком периферии UART, включающий в себя:

1.            Инициализация модуля;

2.            Установка бодовой скорости;

3.            Установка бита четности;

4.            Установка количества стоп-битов;

5.            Установка функции обратного вызова (callback) для приема данных;

6.            Функция передачи массива данных;

Если описанных выше требований недостаточно для обеспечения работоспособности модуля, то другие параметры могут быть переданы на Ваше усмотрение.

Дополнительно:

1.            Блок периферии работает только с прерываниями

2.            Модуль должен иметь возможность работать одновременно с несколькими блоками периферии (USART1, USART2 etc.)

3.            Публичный API должен состоять из одного заголовочного файла (platform-specific исходный файлы не нужны).

 

Шаг 2

На основе Шага 1 написать модуль, который использует публичный API UART (USART) для получения и отправки данных между двумя модулями UART.

Данные для блока 1 (USART1):

1.            Бодовая скорость 9600;

2.            Только получение данных (RX линия активна);

Данные для блока 1 (USART2):

1.            Бодовая скорость 115200;

2.            Только передача данных (TX линия активна);

Особенности выполнения шага 2:

1.            После получения данных через USART1, данные должны быть сохранены и отправлены через USART2;

2.            Отправка данных через USART2 осуществляется блоками по 128 байт. Достаточное количество данных должно быть аккумулировано перед отправкой через USART2;

3.            В момент передачи данных через USART2 возможно получение новых данных от USART1. Новые данные или передаваемые в текущий момент данные не должны быть потерены.

4.            Для проверки функциональности должны быть разработаны тесты. (Выбор типа тестов на Ваше усмотрение).

Общие положения

Общие положения

1.            Модули должны быть написаны на языке C или C++. Предпочтительным является язык С++ или комбинация С/С++.

2.            Дизайн, то как параметры передаются в функции, а также использование конструкций языка важны при оценке решения.

3.            Пожалуйста используйте GitHub для публикации Вашего решения.
